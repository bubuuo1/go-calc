# 로컬 가계부 웹 애플리케이션

## 프로젝트 개요

Supabase 데이터베이스를 사용하는 가계부 웹 애플리케이션입니다.

프론트엔드는 Next.js, React, TypeScript, Tailwind CSS로 구성되어 있으며, 거래 데이터는 Supabase의 `transactions` 테이블에 저장합니다.

## 주요 기능

- 수입 / 지출 거래 추가
- 거래 수정 및 삭제
- 현금 / 카드 결제수단 입력
- 금액 1,000원 단위 콤마 표시
- 날짜 직접 입력 및 작은 달력 레이어 선택
- 카테고리 선택
- 메모 작성
- 거래 검색 및 필터
- 거래 리스트 페이지네이션
- 월간 달력에서 일별 수입 / 지출 확인
- 일별 통계 그래프 페이지
- Supabase 기반 데이터 저장

## 실행 방법

```bash
npm install
npm run dev
```

접속 주소:

- 입력 화면: `http://localhost:3000`
- 통계 화면: `http://localhost:3000/stats`
- 카테고리 분석: `http://localhost:3000/categories`

## Supabase 설정

1. Supabase 프로젝트를 생성합니다.
2. Supabase SQL Editor에서 `supabase/schema.sql` 내용을 실행합니다.
3. `.env.example`을 참고해서 `.env.local`을 만듭니다.

```env
NEXT_PUBLIC_SUPABASE_URL=https://your-project-ref.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-public-key
```

기존 `data/transactions.json` 데이터를 Supabase로 옮기려면 `.env.local` 설정 후 아래 명령을 실행합니다.

```bash
npm run migrate:supabase
```

Vercel에 배포할 때도 같은 환경변수 2개를 Project Settings > Environment Variables에 등록해야 합니다.
